# Greeting Service

> Arrival-greeting generation with pluggable context providers.

## Overview

The Greeting service produces a personalized morning greeting when a known user arrives (via the Presence service's `presence.arrived` event). The greeting is generated by an AI call against the configured profile, with zero tools forced — the greeting prompt can never accidentally trigger broadcasts, announcements, or other side effects.

## Context providers

Other services contribute optional context that the greeting AI may use, via the **`GreetingContextProvider`** capability protocol (`src/gilbert/interfaces/greeting.py`):

```python
@runtime_checkable
class GreetingContextProvider(Protocol):
    @property
    def greeting_context_id(self) -> str: ...

    @property
    def greeting_context_label(self) -> str: ...

    async def greeting_context(self, user_id: str) -> GreetingContext | None: ...
```

A provider returns a labeled prose fact (or `None` to skip), and GreetingService folds the collected facts into the arrival prompt as a single `Available context:` block. **There is no slot ordering** — the AI integrates the facts based on the user-customizable prompt template.

### Bundled providers

- **WeatherService** (id `"weather"`) — current conditions at the configured home location.
- **FeedBriefingService** (id `"briefing"`) — daily news briefing with a same-day "already shown" guard so the user isn't double-briefed when the briefing service also fires its own daily event.
- **HealthService** (id `"health"`) — sleep / steps / weight / resting-HR / flags from the user's health data.

Each provider:
- Implements `greeting_context(user_id)` and never raises (returns `None` on any failure).
- Owns its own format template / settings (e.g., WeatherService owns `weather_hint_template`).
- Advertises `"greeting_context"` in its `ServiceInfo.capabilities`.

### Discovery

GreetingService calls `resolver.get_all("greeting_context")` once after its resolver is set, then iterates the resulting list per arrival to assemble the context block. Providers from any plugin show up automatically — no greeting service code edits required.

### Settings

The greeting settings page renders one toggle per discovered provider, driven by the `greeting.context_providers.list` WS RPC. Toggles persist to the `greeting.enabled_context_providers` config (a `list[str]` of provider ids). `None` (default) means all discovered providers are enabled.

### Custom prompts

`greeting.arrival_greeting_prompt` is a `ConfigParam(ai_prompt=True)` with placeholders `{name}`, `{style_instruction}`, `{context_section}`, `{avoid_section}`. Users can write rules like "always mention the weather if it's extreme" — the AI sees the labeled context block and the prompt's instructions and combines them.

## Trigger and gating

- Subscribes to `presence.arrived` and runs `_greet_user(user_id)` if the current time is inside `[start_hour, cutoff_hour)` in the configured timezone.
- Per-user `asyncio.Lock` (lazily created in `_greeting_locks`) prevents concurrent presence events from double-greeting the same person.
- "Greeted today" persistence in `greeting_state` collection: `last_greeting_date` (today) gates re-greets; `recent_greetings` (last 10) is fed back to the AI as anti-repetition context.
- Startup hook: scheduled `Schedule.once_after(45)` job calls `_greet_already_present` so a Gilbert restart while people are already at the shop doesn't drop their greeting (presence service suppresses events on the first poll).

## AI generation path

`_generate_greeting(name, recent)` calls `AISamplingProvider.complete_one_shot` with `tools_override=[]` (forces zero tool access regardless of profile) and `profile_name=self._ai_profile` (default `"light"`). The prompt mentions the name, the style instruction (if any), and includes the recent-greetings list to avoid repetition. Group greetings (`_generate_group_greeting`) batch the names and produce one combined message instead of stepping over each other.

## Anti-repetition

`recent_greetings` rolls 10 entries; the most recent 7 are fed into the prompt's "Here are your recent greetings — do NOT repeat" section. Persisted per-user under `greeting_state.{user_id}`.

## Tools / slash commands

Single tool `greet` (slash command `/greet <name>`) — calls `_generate_greeting(person_name)` and announces. No multi-target slash group.
